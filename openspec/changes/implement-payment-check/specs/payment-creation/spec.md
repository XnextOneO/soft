## ADDED Requirements

### Requirement: Валидация платежного поручения
Система ДОЛЖНА отправлять данные формы на проверку без создания платежа и собирать payload из всех табов в единую DTO.

#### Scenario: Запрос на проверку
- **WHEN** пользователь нажимает «Проверить» на форме создания платежного поручения
- **THEN** фронтенд отправляет PUT-запрос на временный URL `/payment/payment-instruction/check` с JSON, содержащим `paymentMainDto`, `paymentDetails` (описания `description1`…`description5`), `autoPaymentDetails`, `paymentRuKzIn`, `administrativeData`
- **AND** пустые строковые поля передаются как `""`, пустые числовые — как `0`
- **AND** значения дат сериализуются в формате `YYYY-MM-DD`
- **AND** заблокированные (read-only/disabled) поля включаются с их текущими значениями или дефолтами

#### Scenario: Управление полями ввода
- **WHEN** пользователь редактирует доступные поля в любом табе
- **THEN** введенные значения обновляют единый стейт родительского компонента
- **AND** каждый таб работает как управляемый компонент через `value/onChange`
- **AND** состояние готово для сборки payload без обращения к refs дочерних компонентов
