## ADDED Requirements

### Requirement: Управление подключением AuthProvider через env-флаг
Приложение MUST подключать `AuthProvider` только при разрешающем флаге окружения и по умолчанию сохранять текущее включенное состояние.

#### Scenario: Значение отсутствует или пустое
- **WHEN** переменная `VITE_ENABLE_AUTH_PROVIDER` отсутствует или пуста
- **THEN** приложение подключает `AuthProvider` и выполняет текущую auth-логику

#### Scenario: Провайдер отключен флагом
- **WHEN** `VITE_ENABLE_AUTH_PROVIDER` задана в значение `false` (регистр не учитывается)
- **THEN** приложение не подключает `AuthProvider`
- **AND** остальные провайдеры и рендер маршрутов работают без зависимости от auth-стора

#### Scenario: Провайдер явно включен
- **WHEN** `VITE_ENABLE_AUTH_PROVIDER` задана в значение `true`
- **THEN** приложение подключает `AuthProvider` и использует текущие механизмы авторизации
