# Change: Управление подключением AuthProvider через флаг окружения

## Why
В некоторых окружениях требуется запускать приложение без встроенной авторизации, но текущий `AuthProvider` всегда подключается. Нужен управляемый флаг, при этом дефолт должен сохранять нынешнее поведение.

## What Changes
- Добавить env-флаг `VITE_ENABLE_AUTH_PROVIDER` (по умолчанию включен) для контроля обертки `AuthProvider` в `src/routes/__root.tsx`.
- Изменить загрузку провайдеров: при флаге `false` приложение рендерится без `AuthProvider`, иначе — как сейчас.
- Синхронизировать пример env-файла и дефолтное поведение (fallback к включению при отсутствии значения).

## Impact
- Affected specs: auth-provider
- Affected code: src/routes/__root.tsx, .env, .env.example
